📋 Gemini CLI コマンド一覧

💡 使い方のコツ

各コマンドの右側にある「Copy」ボタンをクリックしてクリップボードにコピーできます。ターミナルに貼り付けてそのまま実行してください。

種別 コマンド・フラグ 説明 主な用途
基本 /help すべてのコマンドとショートカットを一覧表示 迷った時の"コマンド辞書"
/clear 画面と履歴を一括クリア（Ctrl+L 同等） 画面をリセットして見やすく
/quit セッションを終了 GeminiCLI を安全に閉じる
外観 /theme ダーク／ライトなど見た目を変更 長時間作業の目の負担軽減
認証 /auth OAuth ↔ APIキーを切替 個人・組織アカウントの使い分け
編集 /editor 外部エディタを選択・設定 vim や code へ即ジャンプ
統計 /stats トークン使用量や経過時間を表示 API費用・利用状況のチェック
情報 /about CLI とモデルのバージョンを表示 バグ報告や再現手順に便利
バグ報告 /bug GitHub Issue を半自動生成 不具合をすぐ開発元へ共有
ドキュメント /docs 公式ドキュメントをブラウザで開く 詳細仕様を即参照
会話管理 /chat save <タグ> 現在の会話をタグ付きで保存 複数プロジェクトを並行管理
/chat resume <タグ> 保存した会話を復元 作業を継続
/chat list 保存済みタグを一覧表示 どの会話があるか確認
/compress 会話全体を要約に置換しトークン節約 長期セッションの軽量化
メモリ /memory add <テキスト> 永続メモリ(GEMINI.md)へ追加 プロジェクト固有ルールを記憶
/memory show 現在のメモリ内容を表示 AI が何を覚えているか確認
/memory refresh GEMINI.md を再読み込み 外部編集後に内容を反映
ツール /tools 利用可能ツール一覧と説明を表示 どんな自動化が可能か把握
/mcp MCP サーバー状態とツール一覧 自社ホストツールとの連携確認
復元 /restore [ID] ツール実行前の状態にロールバック 誤編集・誤削除の安全網
ファイル挿入 @<パス> 指定ファイル/ディレクトリを読ませる コードレビュー・仕様読込み
シェル実行 !<コマンド> シェルコマンドをその場で実行 画像変換や git 操作を自動化

🚀 起動フラグ

フラグ 説明 使用例
--prompt / -p 非対話モードで一発実行 スクリプトやパイプライン組込
--all-files / -a カレント以下の全ファイルを読込み 大規模リポジトリの要約
--sandbox / -s Docker サンドボックス内で安全実行 破壊的コマンドの隔離
--yolo すべて自動承認（確認ダイアログ無効） 非対話バッチ処理を高速化
--model <名前> 使用モデルを選択（例：flash） 速度・精度を状況で切替

🎯 覚えるべき３つのプレフィックス

/ … CLI の動作や会話をコントロール（メタ命令）
@ … ファイル・ディレクトリを Gemini に読み込ませる
! … ローカルシェルを AI 経由で実行
